# 技術選定ドキュメント

## 1. プロジェクト概要

本プロジェクトは、日本の世界遺産を「知る・訪れる・記録する」ことを目的とした
**フルスタック Web アプリケーション**である。

短期間での開発を前提としつつ、
**実務で通用する技術構成・拡張性・保守性**を重視して技術選定を行った。

---

## 2. 全体アーキテクチャ

| レイヤ     | 技術                         | 役割                 |
| ---------- | ---------------------------- | -------------------- |
| Frontend   | Next.js / TypeScript / React | UI・UX               |
| Styling    | Tailwind CSS                 | デザイン・レイアウト |
| Backend    | FastAPI / Python             | API 提供             |
| ORM        | SQLAlchemy                   | DB 操作              |
| DB         | Supabase（PostgreSQL）       | 業務データ管理       |
| Auth       | Firebase Authentication      | 認証                 |
| Storage    | Firebase Storage             | 画像管理             |
| Payment    | Stripe                       | 決済処理             |
| Payment DB | Cloud Firestore              | 決済・課金情報管理   |
| AI         | OpenAI API                   | 解説生成・旅行提案   |
| Map        | Google Maps Platform         | 地図表示             |
| Infra      | Docker / Docker Compose      | 環境構築             |

---

## 3. フロントエンド

### 技術構成

- 言語：TypeScript
- フレームワーク：Next.js（React）
- スタイリング：Tailwind CSS

### 採用理由

- SSR / SSG による **高速な初期表示・SEO 向上**
- 写真・地図を多用するアプリと相性が良い
- 実務での採用率が高く、学習成果を将来に活かしやすい

---

## 4. バックエンド

### 技術構成

- 言語：Python
- フレームワーク：FastAPI

### 採用理由

- API 開発に特化した軽量フレームワーク
- Pydantic による **型安全な入力バリデーション**
- OpenAPI（Swagger）による **API 仕様の自動生成**
- AI 処理やデータ連携との親和性が高い

---

## 5. ORM

### SQLAlchemy

- FastAPI 公式ドキュメントで利用実績があり、情報量が多い
- PostgreSQL との相性が良く、複雑なクエリにも対応可能
- 将来の機能拡張や構成変更にも柔軟に対応できる

---

## 6. データベース

### Supabase（PostgreSQL）

- JSONB・全文検索など高機能な PostgreSQL を利用可能
- マネージド DB のため運用負荷を軽減
- AI 生成テキストやユーザー行動データなど、
  **将来的に拡張されるデータ構造に対応可能**

---

## 7. 認証・画像管理

### Firebase Authentication

- 認証・ユーザー管理を自前実装せずに済み、短期間で安全な実装が可能
- 実務でも利用頻度が高く、学習価値が高い

### Firebase Storage

- 世界遺産画像やバッジ画像の管理に使用
- SDK を用いた簡単なアップロード・取得が可能
- 認証と連携したアクセス制御により、安全な画像配信を実現

---

## 8. AI 活用

### 基本機能：世界遺産解説生成

- 世界遺産の解説文は **事前に AI で生成し、DB に保存**
- 毎回 API を呼ばない設計により

  - 表示速度の安定
  - コスト削減
  - 回答内容の再現性
    を実現

### プレミアム機能：分断のない旅行体験

- 従来の旅行計画は
  **世界遺産 → アクセス → グルメ → 宿泊** を
  それぞれ別サービスで調べる必要があった
- 本アプリでは AI を活用し、

  - どう行くか
  - どこで食べるか
  - どこに泊まるか
    まで含めた **一括旅行プラン** を提案

- 「調べる手間」をなくし、
  **分断のない旅行体験**を実現することを目指す

---

## 9. 決済

### Stripe

- 実務で利用頻度の高い決済サービス
- 将来的なプレミアム機能提供を見据えて導入

### Cloud Firestore（Stripe 連携用）

- Stripe の決済結果・課金状態・サブスクリプション情報の保存に使用
- 構造が変化しやすい決済イベントデータを柔軟に管理可能
- Firebase Authentication と連携し、ユーザー単位で安全に管理
- 業務データ（世界遺産・訪問履歴）とは分離し、責務を明確化
